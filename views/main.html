WebGL Memory Optimization

Optimizations:

  1. Obj reuse through memory pool for Model-View Matrix
  2. Remove Redundant GL State Machine Calls
  3. Remove Redundant Computation of Matrixcies

Result:

  1. Reduced GL calls from 553 to 150 for each frame
  2. Reduce heap allocation from 5KB to 10B
  3. Less GC pause period. from 5.2ms to 0.19ms

Learned:

  1. Even a simple/vanilla pool can be quite effective.
  2. Less GC, More budget to do crazier effects.
